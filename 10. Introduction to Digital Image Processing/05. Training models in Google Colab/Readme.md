## Step 3: Fueling the Rocket (Switching to GPU)

Before we write a single line of code, we need to upgrade our engine. Training a YOLOv11 object detection model on a standard CPU is like trying to mow a football field with safety scissors—it's going to take forever. We need a **GPU** (Graphics Processing Unit) to do the heavy lifting.

1. Look at the top menu bar and click on **Runtime**.
2. Select **Change runtime type** from the dropdown menu.
3. Under *Hardware accelerator*, switch it from *CPU* to **T4 GPU** (or whichever free GPU Colab provides).
4. Click **Save**. 

Now your notebook is officially backed by a beastly graphics card!

## Step 4: Summoning the Dataset (Roboflow Integration)

Now that our lab is powered up, we need something for our AI to learn from. We will use the Roboflow library to download our annotated images and labels directly into Colab.

Create a new code cell, paste the following snippet, and hit the **Play** button (or press `Shift + Enter`):
## First, install the Roboflow library
``` !pip install roboflow ```

## Connect to your project and download the YOLOv11 formatted dataset
```
from roboflow import Roboflow
rf = Roboflow(api_key="YOUR_API_KEY_HERE") # Replace with your secret Roboflow API key
project = rf.workspace("aranyas-workspace-n3qnr").project("My First Project")
dataset = project.version(1).download("yolov11")
```

## Step 5: Unleashing YOLOv11 (The Training Phase)
The stage is set. The data is ready. It's time to train the brain! We will use the ultralytics library to load up a pretrained YOLOv11 model and fine-tune it on our custom data.

Create one more code cell and run this final block:

```
# Install the Ultralytics package which powers YOLOv11
!pip install ultralytics

from ultralytics import YOLO

# 1. Load a lightweight, speedy YOLOv11 nano model
model = YOLO("yolov11n.pt") 

# 2. Start the training countdown!
results = model.train(
    data="/content/My-First-Project-1/data.yaml",  # Path to the data.yaml file generated by Roboflow
    epochs=50,                                      # How many times the AI reviews the dataset
    imgsz=640,                                      # Standard image resolution size
    batch=16,                                       # How many images it processes simultaneously
    device=0                                        # Explicitly telling it to use our shiny T4 GPU
)
```

As the code execution runs, you'll see real-time updates of loss metrics, precision scores, and maps. Your AI is officially learning how to see!
